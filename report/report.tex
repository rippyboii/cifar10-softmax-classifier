\section{Training Results}

The network was trained on four hyperparameter settings, all with \texttt{n\_batch}=100 and \texttt{n\_epochs}=40. Training used \texttt{data\_batch\_1} (10,000 images), validation was \texttt{data\_batch\_2}, and testing was on \texttt{test\_batch}. Each dimension was normalized using training set statistics, and weights were drawn from a Gaussian ($\sigma=0.01$, seed=42).

\subsection{Configuration 1: $\lambda=0$, $\eta=0.1$}

A learning rate of 0.1 makes training very unstable, the loss swings between 3 and 10 across epochs and never really settles down. There's also a big gap between training accuracy (42\%) and validation accuracy (27\%), suggesting overfitting on top of the instability. The weight templates look like noise, so the network hasn't picked up on any meaningful patterns. Test accuracy lands at 27.70\%, barely above the 10\% random baseline, which confirms that this learning rate is just too high.

\subsection{Configuration 4: $\lambda=1.0$, $\eta=0.001$}

Strong regularization ($\lambda=1.0$) is where things get interesting. Train and validation loss nearly overlap, so overfitting is basically gone, but the loss flatens early (around epoch~15) at ${\sim}1.80$, higher than the ${\sim}1.61$ in Configs~2 and~3, meaning the network underfits. The cost starts high (${\sim}4.0$) due to the penalty on random initial weights, then drops fast as they shrink. The weight templates are now actually readable: airplane picks up blue sky, frog is green, ship has blue water, horse shows a brown body. Test accuracy falls to 37.55\%.

\section{Exercise 2.1: Bonus Improvements}

Four improvements were applied cumulatively on top of the baseline (Config~3):
\begin{enumerate}
  \item \textbf{All training data} : 49k images instead of 10k (1k held out for validation).
  \item \textbf{Horizontal flip augmentation} : each image flipped with $p{=}0.5$ per mini-batch.
  \item \textbf{Step-decay LR} : $\eta \times 0.1$ every 20 epochs.
  \item \textbf{Grid search} : $3{\times}3{\times}3$ over $\eta \in \{0.01, 0.005, 0.001\}$, $\lambda \in \{0.01, 0.05, 0.1\}$, batch $\in \{50, 100, 200\}$.
\end{enumerate}

The biggest gain came from simply using more training data. With augmentation in the mix, the network didn't need as much L2 regularization, so $\lambda{=}0.01$ consistently outperformed $\lambda{=}0.1$. The best setup ended up being $\eta{=}0.005$, $\lambda{=}0.01$, batch size 200, hitting \textbf{42.80\% validation} and \textbf{42.30\% test accuracy}, up from 39.30\% at baseline.

\section{Exercise 2.2: Multiple Binary Cross-Entropy Loss}

\subsection*{Gradient}

Replacing softmax with element-wise sigmoid and using the multiple BCE loss
$l_{\text{BCE}} = -\frac{1}{K}\sum_k [(1-y_k)\log(1-p_k) + y_k\log(p_k)]$,
the gradient w.r.t.\ scores is:
\[
  \frac{\partial l_{\text{BCE}}}{\partial s_k} = \frac{1}{K}(p_k - y_k)
\]
identical in form to the softmax case but scaled by $1/K$.

\subsection*{Results}

Both models trained with batch$=50$, $\lambda{=}0.01$, step decay, flip aug, 49k images. BCE used $\eta{=}0.01$ (scaled by $K$) vs.\ $\eta{=}0.001$ for softmax.
