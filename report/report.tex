\section{Introduction}

This report describes the implementation and evaluation of a single-layer softmax classifier trained on the CIFAR-10 dataset using mini-batch gradient descent. The model maps each 3072-dimensional image vector to a probability distribution over 10 classes via a linear transformation followed by softmax. The cost function combines the mean cross-entropy loss with an L2 regularization term on the weight matrix $W$. Four hyperparameter configurations are explored to study the effect of learning rate and regularization strength.

\subsection*{Improvement 1: All Available Training Data}

Instead of using only \texttt{data\_batch\_1} (10\,000 images), all five training batches are concatenated into 50\,000 images. The last 1\,000 images of \texttt{data\_batch\_5} are reserved as the validation set (1\,000 images), giving 49\,000 training images. Mean and standard deviation are recomputed from the 49k training split. Using five times more data is the single largest driver of accuracy gain for a linear model, as the decision boundaries are estimated from a much richer sample.

\subsection*{Improvement 2: Horizontal Flip Augmentation}

During training, each image in every mini-batch is independently flipped horizontally with probability 0.5. The flip indices are precomputed once using the CIFAR-10 column-vector layout and applied in-place on a copy of the batch; validation and test data are never augmented. Augmentation effectively expands the training distribution, reducing overfitting. Because augmentation already acts as a regularizer, the $\lambda$ value in the grid-search step is kept lower than in the baseline.

\subsection*{Improvement 3: Step-Decay Learning Rate}

The learning rate is multiplied by 0.1 every 20 epochs (\texttt{decay\_factor}=0.1, \texttt{decay\_every}=20). Starting from a higher initial $\eta$ allows faster early progress; the decay then fine-tunes the parameters once the loss is in a flatter region. This is known as \emph{step decay} and is controlled via two new optional keys in the \texttt{GDparams} dictionary, so the existing training loop required no structural changes.

\subsection*{Improvement 4: Grid Search}

A $3\times3\times3$ grid was searched over $\eta \in \{0.01, 0.005, 0.001\}$, $\lambda \in \{0.01, 0.05, 0.1\}$, and \texttt{n\_batch} $\in \{50, 100, 200\}$ (27 combinations). All runs used flip augmentation and step decay ($\times$0.1 at epoch~20). Validation accuracy after 40 epochs was the selection criterion.

The best configuration found was $\eta{=}0.001$, $\lambda{=}0.01$, \texttt{n\_batch}$=50$, achieving 42.70\% validation accuracy. Two clear trends emerged: (1)~reducing regularization from $\lambda{=}0.1$ to $\lambda{=}0.01$ consistently improved results across all batch sizes, because horizontal flip augmentation already provides implicit regularization, making strong L2 redundant; (2)~$\eta{=}0.001$ with step decay outperformed $\eta{=}0.005$, which converges too fast in the first 20 epochs and benefits less from the LR drop at epoch~20. Smaller mini-batches (50) gave a marginal edge over larger ones at this learning rate, consistent with the SGD generalization literature.

\section{Discussion}

\subsection*{Effect of the Learning Rate}

The comparison between Configs~1 and~2 demonstrates the critical importance of selecting an appropriate learning rate. A learning rate of 0.1 is far too large for this problem, causing the optimization to overshoot minima and oscillate erratically. Reducing it to 0.001 produces stable, monotonic convergence. This illustrates that while larger learning rates can in principle speed up training, they risk divergence if they exceed the curvature of the loss surface.

\subsection*{Effect of Regularization}

Comparing Configs~2, 3 and~4 reveals the regularization tradeoff. Without regularization ($\lambda=0$), the model overfits: training loss continues to decrease while validation loss stagnates, and the weight templates are noisy. Moderate regularization ($\lambda=0.1$) slightly reduces the generalization gap and produces marginally better test accuracy (39.30\% vs 39.21\%). Strong regularization ($\lambda=1.0$) eliminates overfitting entirely but constrains the model so heavily that it underfits, lowering test accuracy to 37.55\%. The most notable benefit of strong regularization is the interpretability of the weight templates, which become recognizable class prototypes. This reflects the fact that L2 regularization penalizes large weights, forcing the model to rely on smooth, low-frequency features rather than fitting noise in the training data.

\section{Exercise 2.2: Multiple Binary Cross-Entropy Loss}

\subsection*{Setup}

Instead of softmax, each score $s_k$ is mapped independently to a probability via the sigmoid function
\[
  p_k = \sigma(s_k) = \frac{1}{1+e^{-s_k}}, \quad k=1,\ldots,K
\]
and the network is trained with $K$ independent binary cross-entropy losses:
\[
  l_{\text{BCE}}(\mathbf{x},\mathbf{y})
  = -\frac{1}{K}\sum_{k=1}^{K}
    \bigl[(1-y_k)\log(1-p_k) + y_k\log(p_k)\bigr].
\]

\subsection*{Gradient Derivation}

We compute $\partial l_{\text{BCE}}/\partial s_k$ using the chain rule.
Since $\sigma'(s) = \sigma(s)\bigl(1-\sigma(s)\bigr) = p_k(1-p_k)$:

\begin{align*}
  \frac{\partial l_{\text{BCE}}}{\partial s_k}
  &= -\frac{1}{K}\left[
      (1-y_k)\cdot\frac{-\sigma'(s_k)}{1-p_k}
      + y_k\cdot\frac{\sigma'(s_k)}{p_k}
    \right] \\
  &= -\frac{1}{K}\left[
      -(1-y_k)\,p_k + y_k(1-p_k)
    \right] \\
  &= -\frac{1}{K}\left[-p_k + y_k\right]
   = \frac{1}{K}(p_k - y_k).
\end{align*}

In matrix form over a mini-batch of size $n$:
\[
  \frac{\partial l_{\text{BCE}}}{\partial \mathbf{s}}
  = \frac{1}{K}(\mathbf{P} - \mathbf{Y}),
\]
which has the same structure as the softmax gradient $\mathbf{P}-\mathbf{Y}$
but scaled by $1/K$.
The gradients w.r.t.\ $W$ and $\mathbf{b}$ are therefore identical in form:
\[
  \frac{\partial J}{\partial W}
  = \frac{1}{n}\,\frac{1}{K}(\mathbf{P}-\mathbf{Y})\mathbf{X}^\top + 2\lambda W,
  \qquad
  \frac{\partial J}{\partial \mathbf{b}}
  = \frac{1}{n}\,\frac{1}{K}(\mathbf{P}-\mathbf{Y})\mathbf{1}.
\]

\subsection*{Results}

Because the $1/K$ factor shrinks gradient magnitudes by a factor of $K=10$,
the learning rate was increased by the same factor ($\eta=0.01$ for BCE vs.\
$\eta=0.001$ for softmax). Both models were trained with the same
setup: \texttt{n\_batch}=50, $\lambda=0.01$, step decay $\times 0.1$ at
epoch~20, flip augmentation, 49k training images.
